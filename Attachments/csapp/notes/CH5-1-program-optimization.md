# CH5-1 程序优化的思路

> CMU 15-213: Introduction to Computer Systems, 10th Lecture, Oct. 1, 2015

## 概述

程序优化是计算机系统中提升软件性能的重要手段。本章涵盖：

- 程序优化的概念
- 常见的程序优化思路
- 编译优化 —— 程序优化的自动化方法
- 编译优化的局限

## 影响程序性能的因素

- **软件**：算法、开发框架、编程语言、编程技巧、编译
- **硬件**：指令集、指令集的实现（x86 变种、arm 变种）、芯片制程

性能不只是算法中的渐进复杂度，渐进复杂度中的常量因子也很重要。代码的编写方式很容易导致 10 倍的性能差异。

## 面向计算的性能优化方法

### 1. 删除无用的计算

有些计算没有副作用，可以直接删除：

- 不影响函数返回值
- 不影响全局变量
- 不影响系统环境（如系统调用）
- 不影响别的线程/进程

### 2. 完成常量的计算

如果表达式中的全部或部分成分是**编译时常量**，则可以提前完成这些计算，减少运行时的计算开销。

### 3. 消除重复的计算

**公共子表达式（Common Subexpressions）**：在代码的多个位置重复出现且值不会发生变化的表达式。

```c
// 优化前：4 个乘法
up = val[(i-1)*n + j];
down = val[(i+1)*n + j];
left = val[i*n + j-1];
right = val[i*n + j+1];

// 优化后：1 个乘法
long inj = i*n + j;
up = val[inj - n];
down = val[inj + n];
left = val[inj - 1];
right = val[inj + 1];
```

**循环不变式（Loop Invariant）**：将不变式提到循环外计算，复用它。

```c
// 优化前：n*i 在循环内反复计算
void set_row(double *a, double *b, long i, long n) {
    long j;
    for (j = 0; j < n; j++)
        a[n*i + j] = b[j];
}

// 优化后：循环不变式外提
void set_row(double *a, double *b, long i, long n) {
    long j;
    int ni = n*i;
    for (j = 0; j < n; j++)
        a[ni + j] = b[j];
}
```

### 4. 简化复杂的计算

使用更快的等价表达式替换：`x * 2^k` -> `x << k`

在循环内部，用便宜的指令替换昂贵的指令：

```c
// 优化前
for (i = 0; i < n; i++) {
    int ni = n*i;
    for (j = 0; j < n; j++)
        a[ni + j] = b[j];
}

// 优化后：用增量加法替代乘法
int ni = 0;
for (i = 0; i < n; i++) {
    for (j = 0; j < n; j++)
        a[ni + j] = b[j];
    ni += n;
}
```

### 5. 减少控制的开销（"降低管理税"）

代码中大量的控制结构（循环、函数、多态等）会带来控制开销。

**函数调用的管理税**：递归 vs 循环

```c
// 递归版本：函数体需要 11 条指令
long fact(long n) {
    if (n < 2) return 1;
    else return n * fact(n-1);
}

// 循环版本：循环体 3 条指令
long factLoop(long n) {
    long acc = 1;
    for(long i = 2; i < n+1; ++i)
        acc = acc * n;
    return acc;
}
```

**循环结构的管理税**：循环展开、循环融合

```c
// 循环展开
for (int i = 0; i+1 < N; i+=2) {
    a[i] = b[i] * c[i];
    a[i+1] = b[i+1] * c[i+1];
}

// 循环融合（2*N 次管理税 -> N 次）
for (int i = 0; i < N; i++) {
    a[i].x = b[i].x;
    a[i].y = b[i].y;
}
```

**解释执行的管理税**：Java、Python 等采用解释模式。JIT（Just In Time）编译作为解释模式的性能优化，将热点函数编译为机器码执行。

## 编译优化

编译器提供了从程序到机器的高效映射：

- **机器无关优化**：消除无用计算、常量计算、消除重复计算、简化复杂计算、减少控制开销
- **机器相关优化**：寄存器分配、缓存行对齐、缓存局部性、指令选择、指令调度

### 数据流分析（Data-Flow Analysis）

基于函数的控制流图，计算每条语句前后的静态属性（不变式）。

**基本思路**：

1. 处理每条语句：该语句修改了哪个变量，访问了哪个变量
2. 处理顺序结构：如果 s1 修改了 x，且顺序后继上的 s2 也修改了 x，则 s1 的修改从 s2 开始失效
3. 处理分支结构：如果所有前驱路径中有 k 个语句修改了 x，则这 k 个语句的修改都可能生效
4. 处理循环结构：需要迭代求解

## 编译优化的局限

### 拦路虎 1：内存别名（Memory Aliasing）

两个不同的内存引用指向同一个内存地址时，编译器难以优化。

```c
// sum_rows1：每次迭代都写回内存（3 次访存）
void sum_rows1(double *a, double *b, long n) {
    long i, j;
    for (i = 0; i < n; i++) {
        b[i] = 0;
        for (j = 0; j < n; j++)
            b[i] += a[i*n + j];
    }
}

// sum_rows2：使用局部变量（1 次访存）
void sum_rows2(double *a, double *b, long n) {
    long i, j;
    for (i = 0; i < n; i++) {
        double val = 0;
        for (j = 0; j < n; j++)
            val += a[i*n + j];
        b[i] = val;
    }
}
```

**解决方法**：使用局部变量而非指针进行累计，提示编译器无需考虑别名问题。也可以使用 `__restrict` 关键字。

### 拦路虎 2：函数调用

编译器很难确定函数是否有副作用，遇到函数调用就作保守分析。

**示例**：`strlen` 在循环中反复调用

```c
// lower1：平方级复杂度
void lower1(char *s) {
    size_t i;
    for (i = 0; i < strlen(s); i++)
        if (s[i] >= 'A' && s[i] <= 'Z')
            s[i] -= ('A' - 'a');
}

// lower2：线性级复杂度（手动将 strlen 外提）
void lower2(char *s) {
    size_t i;
    size_t len = strlen(s);
    for (i = 0; i < len; i++)
        if (s[i] >= 'A' && s[i] <= 'Z')
            s[i] -= ('A' - 'a');
}
```

**解决方法**：内联函数、程序员手动优化。

## 总结

面向计算的性能优化五大思路：

1. **删除无用的计算** —— 不影响结果的计算直接删除
2. **完成常量的计算** —— 编译时提前完成常量计算
3. **消除重复的计算** —— 存储并复用计算结果
4. **简化复杂的计算** —— 用更快的等价表达式替换
5. **减少控制的开销** —— 融合带控制的计算，降低管理成本比例
