## 一、二进制小数

### 定点表示

二进制小数点右侧的位表示 2 的负幂次：

```
b_i b_{i-1} ... b_1 b_0 . b_{-1} b_{-2} ... b_{-j}

值 = Σ b_k × 2^k
```

### 示例

| 十进制 | 二进制 |
|--------|--------|
| 5 3/4 | `101.11₂` |
| 2 7/8 | `010.111₂` |
| 1 7/16 | `001.0111₂` |

**规律**：
- 右移 1 位 = 除以 2（unsigned）
- 左移 1 位 = 乘以 2
- `0.111111...₂` 刚好小于 1.0：`1/2 + 1/4 + 1/8 + ... → 1.0`，记作 `1.0 – ε`

### 表示限制

**限制 #1 — 精度有限**：只能精确表示 `x/2^k` 形式的有理数

| 有理数 | 二进制表示 |
|--------|-----------|
| 1/3 | `0.0101010101[01]...₂` |
| 1/5 | `0.001100110011[0011]...₂` |
| 1/10 | `0.0001100110011[0011]...₂` |

> 这就是为什么 `0.1 + 0.2 ≠ 0.3` —— 0.1 和 0.2 在二进制中都是无限循环小数！

**限制 #2 — 范围有限**：固定位数 w 下，整数部分和小数部分的位数分配决定了表示范围

## 二、IEEE 754 浮点数标准

### 背景

- 1985 年建立，通用浮点算术标准
- 在此之前各芯片厂商使用自己的标准
- 所有主流 CPU 都支持
- 由数值分析工程师主导定义（而非硬件工程师）

### 表示格式

基本公式（三种精度统一）：

$$v = (–1)^s \times M \times 2^E$$

| 字段 | 含义 |
|------|------|
| **s** | 符号位，决定正负 |
| **exp** | 指数字段，编码 E（但不等于 E） |
| **frac** | 小数字段，编码 M（但不等于 M） |

### 三种精度

```
单精度 (float):    s:1  exp:8   frac:23   总计 32 bits
双精度 (double):   s:1  exp:11  frac:52   总计 64 bits
扩展精度 (Intel):  s:1  exp:15  frac:63/64 总计 80 bits
```

### 三种表示类型

#### 1. 规格化 (Normalized) — `exp ≠ 000...0 且 exp ≠ 111...1`

- **指数编码**：`E = exp – Bias`，其中 `Bias = 2^{k-1} – 1`
  - 单精度：Bias = 127（exp: 1~254, E: -126~127）
  - 双精度：Bias = 1023（exp: 1~2046, E: -1022~1023）
- **尾数**：隐含前导 1，`M = 1.xxx...x₂`
  - `frac = 000...0` → M = 1.0（最小值）
  - `frac = 111...1` → M = 2.0 – ε（最大值）
- 表示的最小绝对值：`1.0 × 2^{–126} ≈ 1.2 × 10^{–38}`

**示例**：`float F = 15213.0`

```
15213₁₀ = 11101101101101₂
        = 1.1101101101101₂ × 2¹³
  M = 1.1101101101101₂
  frac = 1101101101101 0000000000₂
  E = 13, Bias = 127, Exp = 140 = 10001100₂

结果: 0 10001100 11011011011010000000000
      s   exp           frac
```

**机器表示对比**：
```
15213  (int):   在 gdb 中用 x/4tb 查看
15213.0 (float): 与 int 版本完全不同
```

#### 2. 非规格化 (Denormalized) — `exp = 000...0`

- 用于表示接近 0 的小数
- **指数**：`E = 1 – Bias`（而非 `E = 0 – Bias`）
- **尾数**：隐含前导 0，`M = 0.xxx...x₂`
- 两种情况：
  - `exp = 0, frac = 0` → 表示 **0**（有 +0 和 –0）
  - `exp = 0, frac ≠ 0` → 接近 0 的小数，**均匀分布**
- 最大绝对值：`(1.0 – 2^{-23}) × 2^{-126} = 2^{-126} – 2^{-149}`
- 比规格化最小值还小 `2^{-149}`

#### 3. 特殊值 — `exp = 111...1`

| 条件 | 含义 | 说明 |
|------|------|------|
| `frac = 000...0` | ∞ (无穷) | `1.0/0.0 = +∞`, `–1.0/–0.0 = +∞`, `1.0/–0.0 = –∞` |
| `frac ≠ 000...0` | NaN (非法数) | `sqrt(–1)`, `∞ – ∞`, `∞ × 0` |

### 可视化编码

```
      +∞    –∞    –0      +Denorm  +Normalized    –Denorm  –Normalized    +0   NaN
      |     |     |       |        |              |        |              |    |
<-----|-----|-----|-------|--------|--------------|--------|--------------|----|----->
```

## 三、浮点数的例子与特性

### 8 位迷你浮点 (FP8)

```
s:1  exp:4  frac:3   Bias = 7
```

**值分布表（正数部分）：**

| s | exp | frac | E | Value |
|---|-----|------|---|-------|
| 0 | 0000 | 000 | -6 | 0 |
| 0 | 0000 | 001 | -6 | 1/8 × 1/64 = 1/512 |
| 0 | 0000 | 010 | -6 | 2/8 × 1/64 = 2/512 |
| ... | ... | ... | ... | ... |
| 0 | 0000 | 111 | -6 | 7/8 × 1/64 = 7/512 |
| 0 | 0001 | 000 | -6 | 8/8 × 1/64 = 8/512 |
| ... | ... | ... | ... | ... |
| 0 | 0111 | 000 | 0 | 1 |
| 0 | 0111 | 001 | 0 | 9/8 |
| ... | ... | ... | ... | ... |
| 0 | 1110 | 111 | 7 | 240 |
| 0 | 1111 | 000 | n/a | ∞ |

**间隔规律**：靠近 0 时采样点更稠密（非规格化均匀分布 → 规格化指数分布）

### 6 位浮点 (FP6) — 练习

```
s:1  exp:3  frac:2   Bias = 3
```

| 类型 | 数量 | 范围 |
|------|:----:|------|
| 非规格化 | 4 个 | 0, 1/16, 2/16, 3/16 |
| 规格化 | 24 个 | 4/16 ~ 14 |
| 特殊值 | 2 个 | inf + NaN |

**不可表示的整数范围**：[-16, 15] 中缺少数值如 9, 11, 13, 15

> 问：为什么不用均匀采样？
> 答：对数分布的相对误差更均匀，适合科学计算

### 浮点数比较的特性

可以**近似复用无符号整数比较逻辑**：
1. 必须先比较符号位
2. 必须考虑 –0 = 0
3. NaNs 比较结果棘手
4. 其余情况可等同无符号整数比较
   - 非规格化数 < 规格化数（exp 编码更大）
   - 规格化数 < 无穷（exp 编码更大）

> 好处：不需要专门设计复杂的浮点比较硬件

### 拓展：LLM 中的浮点应用

**大模型基本结构**：
- GPT-3 175B：96 层 → 约 192 个线性 + 激活组合
- LLaMA 2 70B：70 层 → 约 140 个线性 + 激活组合
- 核心计算量：`4 × L × d_model × d_ff`

**量化 (Quantization)**：
- 将 float32 转 INT8 计算，节省计算和内存
- 均匀量化（INT4/INT8）会显著劣化模型性能
- **Microsoft ZeroQuant-FP** (2023)：激活值用 FP8 量化，参数用 FP4
- **DeepSeek** 使用自研浮点格式

## 四、舍入与运算

### 基本思路

```
x +f y = Round(x + y)
x ×f y = Round(x × y)
```

1. 计算出精确结果
2. 根据浮点精度进行舍入
3. 处理溢出：
   - overflow：变成全 1，表示 ∞
   - underflow：变成全 0，表示 0

### 舍入模式

| 模式 | $1.40 | $1.60 | $1.50 | $2.50 | –$1.50 |
|------|:-----:|:-----:|:-----:|:-----:|:------:|
| 靠零舍入 (int div) | $1 | $1 | $1 | $2 | –$1 |
| 向下舍入 (–∞) | $1 | $1 | $1 | $2 | –$2 |
| 向上舍入 (+∞) | $2 | $2 | $2 | $3 | –$1 |
| **偶数舍入 (默认)** | **$1** | **$2** | **$2** | **$2** | **–$2** |

### 偶数舍入 (Round-To-Even)

- **默认模式**，其他模式在统计上有偏
- 规则：如果需要舍去的部分刚好是一半，则让保留的最低位为偶数

**十进制示例**（保留 2 位小数）：
| 原值 | 舍入结果 | 原因 |
|------|:--------:|------|
| 7.8949999 | 7.89 | 小于一半 |
| 7.8950001 | 7.90 | 大于一半 |
| 7.8950000 | 7.90 | 刚好一半，保留位是奇数→入 |
| 7.8850000 | 7.88 | 刚好一半，保留位是偶数→舍 |

**二进制偶数舍入**（保留到 1/4，小数点后 2 位）：
- 二进制"偶数" → 最低保留位为 0
- 二进制"一半" → 舍去的小数 = `100...₂`

| 十进制 | 二进制 | 舍入后 | 动作 |
|:------:|:------:|:------:|:----:|
| 2 3/32 | `10.00011₂` | `10.00₂` | <1/2 ↓ |
| 2 3/16 | `10.00110₂` | `10.01₂` | >1/2 ↑ |
| 2 7/8 | `10.11100₂` | `11.00₂` | =1/2 ↑(向上) |
| 2 5/8 | `10.10100₂` | `10.10₂` | =1/2 ↓(向下) |

### 浮点乘法

```
(–1)^s1 × M1 × 2^E1  ×  (–1)^s2 × M2 × 2^E2

精确结果:  (–1)^s × M × 2^E
  s = s1 XOR s2
  M = M1 × M2
  E = E1 + E2
```

**修正**：
- 若 M ≥ 2，将 M 右移（同步增加 E）
- 若 E 超出范围 → overflow（全 1，表示无穷）
- 将 M 调整到 frac 规定的精度

### 浮点加法

```
(–1)^s1 × M1 × 2^E1 + (–1)^s2 × M2 × 2^E2   (假设 E1 > E2)

精确结果:  (–1)^s × M × 2^E
  1. 对齐指数: E = max(E1, E2)，小指数向大指数看齐
  2. 有符号相加
```

**修正**：
- 若 M ≥ 2，右移（增加 E）
- 若 M < 1，左移 k 位（E 减去 k）
- 若 E 超出范围 → overflow / underflow
- 将 M 调整到 frac 规定精度

### 浮点加法的数学特性

| 性质 | 满足? | 说明 |
|:----|:-----:|------|
| 封闭性 | ✅ | 但可能生成 ∞ 或 NaN |
| 交换律 | ✅ | |
| **结合律** | **❌** | `(3.14+1e10)-1e10 = 0` ≠ `3.14+(1e10-1e10) = 3.14` |
| 存在零元 | ✅ | |
| 存在逆元 | ✅ | 除 inf 和 NaN 外 |
| 单调性 | ✅ | 除 inf 和 NaN 外 |

### 浮点乘法的数学特性

| 性质 | 满足? | 说明 |
|:----|:-----:|------|
| 封闭性 | ✅ | 但可能生成 ∞ 或 NaN |
| 交换律 | ✅ | |
| **结合律** | **❌** | `(1e20×1e20)×1e-20 = ∞` ≠ `1e20×(1e20×1e-20) = 1e20` |
| 存在单位元 | ✅ | |
| **分配律** | **❌** | `1e20×(1e20-1e20) = 0.0` ≠ `1e20×1e20–1e20×1e20 = NaN` |
| 单调性 | ✅ | 除 inf 和 NaN 外 |

### GRS 舍入（浮点构造步骤）

```
1.BBG RXXX
G 位 - Guard bit:      保留结果的最低位
R 位 - 1st bit removed: 第一个被移除的位
S 位 - Sticky:          剩余位的 OR
```

**向上舍入的条件**：
- R=1, S=1 → >0.5，向上舍入
- R=1, S=0 → 靠偶数舍入（G=1 时向上）

### 完整浮点构造流程

1. **规范化** (Normalize)：左移使第一位为 1，同步减小指数
2. **舍入** (Round)：根据 GRS 位舍入到 frac 精度
3. **后续规范化** (Postnormalize)：处理舍入导致的进位溢出

**示例**：将 8 位无符号数转为 FP8

| 原值 | 规范化后 | GRS | 舍入结果 | 调整后 | 最终值 |
|:----:|:--------:|:---:|:--------:|:-----:|:-----:|
| 128 | 1.0000000×2⁷ | 000 | 1.000 | — | 128 |
| 13 | 1.1010000×2³ | 100 | 1.101 | — | 13 |
| 17 | 1.0001000×2⁴ | 010 | 1.000(舍) | — | 16 |
| 19 | 1.0011000×2⁴ | 110 | 1.010(入) | — | 20 |
| 138 | 1.0001010×2⁷ | 011 | 1.001(入) | — | 144 |
| 63 | 1.1111100×2⁵ | 111 | 10.000(入) | 1.000×2⁶ | 64 |

## 五、C 语言中的浮点数

### 精度类型

```c
float  f;  // 单精度 32 位
double d;  // 双精度 64 位
```

### 类型转换

| 转换 | 特点 |
|:----|:-----|
| `double/float → int` | 截断小数部分（靠零舍入）；范围外或 NaN → TMin（未定义行为） |
| `int → double` | **精确**，无损（int ≤ 53 bit） |
| `int → float` | 根据舍入模式进行**舍入** |

### 浮点奇怪现象 (Puzzles)

```c
int x = ...;
float f = ...;
double d = ...;
// 假设 d 和 f 均不是 NaN
```

| 表达式 | 恒真? | 原因 |
|:-------|:-----:|:-----|
| `x == (int)(float)x` | ❌ | 舍入或溢出 |
| `x == (int)(double)x` | ✅ | |
| `f == (float)(double)f` | ✅ | |
| `d == (double)(float)d` | ❌ | 舍入或溢出 |
| `f == -(-f)` | ✅ | |
| `2/3 == 2/3.0` | ❌ | 左侧 = 0（整数除法），右侧 ≈ 0.666... |
| `d < 0.0 ⇒ (d*2) < 0.0` | ✅ | |
| `d > f ⇒ -f > -d` | ✅ | |
| `d*d >= 0.0` | ✅ | |
| `(d+f)-d == f` | ❌ | 溢出或舍入导致精度丢失 |

### 重要数值边界

| 描述 | exp | frac | 数值 |
|:----|:---:|:----:|:-----|
| Zero | 00...00 | 00...00 | 0.0 |
| 最小正非规格化 | 00...00 | 00...01 | 单 ≈ 1.4×10⁻⁴⁵, 双 ≈ 4.9×10⁻³²⁴ |
| 最大非规格化 | 00...00 | 11...11 | 单 ≈ 1.18×10⁻³⁸, 双 ≈ 2.2×10⁻³⁰⁸ |
| 最小正规化 | 00...01 | 00...00 | 略大于最大非规格化 |
| 1.0 | 01...11 | 00...00 | 1.0 |
| 最大正规化 | 11...10 | 11...11 | 单 ≈ 3.4×10³⁸, 双 ≈ 1.8×10³⁰⁸ |

## 总结

- IEEE 浮点数表示形式：**M × 2^E**
- 有了标准就可以推理浮点运算性质，不用管具体实现
- 遵循"先算精确结果，再舍入处理"的模式
- **跟现实算术不一样**：不满足结合律和分配律
- 给编译器和数值计算编程带来困难

## 例题：不同浮点格式之间的转换

> **FP6 格式 A** (1-2-3)：阶码 2 位，小数 3 位，Bias = 1
> **FP6 格式 B** (1-3-2)：阶码 3 位，小数 2 位，Bias = 3

给定 FP6 A 表示 `0 10 111`：

1. 数值 = `1.111 × 2¹ = 11.11₂ = 3.75₁₀`
2. 转 FP6 B：
   - 小数保留 2 位：`1.111 → 10.00`（偶舍入）
   - 指数调整：`10.00 × 2¹ → 1.000 × 2²`
   - 阶码表示（Bias=3）：`0 101 00` → 数值 = **4.0**
3. 从 B 转回 A：
   - 小数恢复 3 位：`1.000`
   - 阶码 Bias=1：`0 11 000`
   - **阶码为全 1** → 溢出为无穷大 ∞
