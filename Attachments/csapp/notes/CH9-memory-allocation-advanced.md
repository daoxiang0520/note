# 动态内存分配：高级知识

## 跟踪空闲块的方法

| 方法 | 描述 |
|------|------|
| 隐式列表 | 用长度链接所有内存块 |
| 显式列表 | 用指针在空闲块中建立链表 |
| 分离式空闲列表 | 按照不同的大小建立不同的空闲列表 |
| 按大小排序的块 | 使用平衡树（红-黑树），用长度作为键 |

## 显式空闲列表

维护空闲 blocks 内存块列表，而不是所有内存块列表。使用有效负载区域保存前向/后向指针。

```
[头部: size|a] [前一个] [下一个] [有效负载] [尾部: size|a]
```

### 释放策略

| 策略 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| LIFO | 插入列表开始位置 | 简单且时间恒定 | 碎片更多 |
| FIFO | 插入列表结束位置 | 简单且时间恒定 | 碎片更多 |
| 按地址排序 | 保持地址顺序 | 碎片化更少 | 需要搜索 |

### 释放时的合并场景

- 场景 1：前后都已分配，直接插入列表
- 场景 2：后空闲，合并后插入列表
- 场景 3：前空闲，合并前插入列表
- 场景 4：前后都空闲，三者合并

### 实现技巧：循环双向链表

使用单一数据结构支持多种方法（首次适配 vs 下一次适配、LIFO vs FIFO）。

### 显式列表总结

- **与隐式列表相比**：分配时间复杂度与空闲块数（而非所有块数）成线性关系
- **缺点**：需要额外空间（每个空闲块需两个附加字）

## 分离式空闲列表 (Seglist)

有若干空闲列表，每个对应一个**大小类**。

### 常见大小类选择

- 每个小尺寸一个类（16, 32, 48, 64, ...）
- 某些时候使用 2 的幂：[16] [32-48] [64-inf]

### 分配算法

1. 搜索适合大小的空闲列表（首次适配）
2. 如果找到：分割并放入适当列表
3. 如果没找到：尝试下一个更大类
4. 如果都没找到：用 `sbrk()` 请求额外堆内存

### Seglist 分配器的优点

- **更高的吞吐量**：2 的幂大小类的 log 时间复杂度 vs 线性
- **更好的内存利用率**：分离式列表的首次适配搜索相当于整个堆的最佳适配搜索

## 内存相关的风险与陷阱

### 1. 间接引用错误指针

```c
int val;
scanf("%d", val);  // 应该是 &val
```

### 2. 读取未初始化的内存

```c
int *y = malloc(N * sizeof(int));
// y[i] 未初始化就使用
```

### 3. 写覆盖内存

```c
int **p;
p = malloc(N * sizeof(int));  // 应该是 sizeof(int*)
```

### 4. 缓冲区溢出

```c
char s[8];
gets(s);  // 读取 "123456789" 会溢出
```

### 5. 大小差一错误

```c
p = malloc(N * sizeof(int*));
for (i = 0; i <= N; i++)  // 应该是 < N
    p[i] = malloc(M * sizeof(int));
```

### 6. 错误理解指针运算

```c
int *search(int *p, int val) {
    while (p && *p != val)
        p += sizeof(int);  // 应该是 p++
    return p;
}
```

### 7. 引用指针而不是指向的对象

```c
*size--;  // 等价于 size--，应该是 (*size)--
```

### 8. 引用不存在的变量

```c
int *foo() {
    int val;
    return &val;  // 局部变量在函数返回后不存在
}
```

### 9. 多次释放内存块

```c
free(x);
free(x);  // 重复释放
```

### 10. 引用已释放内存块

```c
free(x);
y[i] = x[i]++;  // x 已被释放
```

### 11. 未能释放内存块（内存泄漏）

```c
foo() {
    int *x = malloc(N * sizeof(int));
    return;  // 忘记 free(x)
}
```

### 12. 仅释放数据结构的一部分

```c
struct list *head = malloc(sizeof(struct list));
// 创建链表...
free(head);  // 只释放了头节点
```

## 内存漏洞处理方法

| 工具 | 用途 |
|------|------|
| **gdb** | 发现错误指针的间接引用 |
| **valgrind** | 运行时检查每个引用（错误指针、写覆盖等） |
| **glibc malloc MALLOC_CHECK_** | 包含检查代码 |
| **垃圾回收** | 自动回收不再使用的内存 |
