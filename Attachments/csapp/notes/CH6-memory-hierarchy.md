# CH6 存储器层次结构：存储层次与存储技术

> CMU 15-213: Introduction to Computer Systems

## 概述

现代 CPU 与内存之间存在巨大的速度差距，存储器层次结构通过利用局部性原理来弥合这一性能鸿沟。

## 内存读写事务

### 读事务

```
movq A, %rax  # 加载操作
```

1. CPU 将地址 A 放到内存总线上
2. 主存读到 A，检索到字 x，再把 x 放到总线上
3. CPU 从总线读到字 x，拷贝到寄存器 %rax

### 写事务

```
movq %rax, A  # 存储操作
```

1. CPU 将地址 A 放到总线上
2. CPU 将数据字 y 放到总线上
3. 主存从总线读到数据字 y，存储到地址 A

## "内存墙"（Von Neumann 瓶颈）

计算与数据存储设备之间存在巨大的性能差异。三种应对方法：

1. **构造存储层次结构**（本课程重点）
2. **暂时计算别的任务**（并行计算）
3. **消除数据搬运**（存内计算）

## 局部性原理（Principle of Locality）

许多程序倾向于使用与其最近使用过的地址相近或相同的地址来访问数据和指令。

### 时间局部性（Temporal Locality）

最近被访问过的对象很可能在不久的将来再次被访问。应尽量让重复访问的对象长驻小容量的快速存储。

### 空间局部性（Spatial Locality）

地址相邻的对象往往在时间上被集中地连续访问。应尽量将相邻对象预取到小容量的快速存储。

### 局部性示例

```c
long sum = 0;
for (int i = 0; i < n; i++)
    sum += a[i];
return sum;
```

- 访问数据 sum：**时间局部性**（每次迭代都访问）
- 访问数组 a[i]：**空间局部性**（步长为 1 的顺序访问）

### 行优先 vs 列优先

```c
// 行优先遍历（良好的局部性）- 步长为 1
int sum_array_rows(int a[M][N]) {
    int i, j, sum = 0;
    for (i = 0; i < M; i++)
        for (j = 0; j < N; j++)
            sum += a[i][j];
    return sum;
}

// 列优先遍历（差的局部性）- 步长为 N
int sum_array_cols(int a[M][N]) {
    int i, j, sum = 0;
    for (j = 0; j < N; j++)
        for (i = 0; i < M; i++)
            sum += a[i][j];
    return sum;
}
```

数组在内存中按**行优先顺序**（row-major order）存储。

## 存储器层次结构

### 层次结构示例

| 层级 | 名称 | 技术 | 管理方式 |
|------|------|------|----------|
| L0 | 寄存器 | 编译器 | 编译器 |
| L1 | L1 缓存 (SRAM) | 硬件 | 硬件 |
| L2 | L2 缓存 (SRAM) | 硬件 | 硬件 |
| L3 | L3 缓存 (SRAM) | 硬件 | 硬件 |
| L4 | 主存 (DRAM) | 硬件+OS | OS |
| L5 | 本地磁盘 | OS | OS |
| L6 | 远程存储 | Web 服务器 | Web 浏览器 |

### 核心理念

对于每一层 k，第 k 层上更快、更小的设备作为第 k+1 层上更大、更慢设备的缓存。存储层次结构构建了一个庞大的存储池——其成本接近底层廉价存储，却能以顶层高速存储的速度向程序提供数据。

### 工作集和局部性

**工作集**：程序当前"正在使用"的数据集合。局部性有助于减小工作集的实际规模。当工作集能够完全容纳在快速存储中时，快速存储的效果最佳。

### Cache vs 内存

- **缓存**对软件是透明的（不可见的），由硬件根据 load/store 操作管理
- **内存**对软件是可见的，指令可直接通过内存地址进行寻址

## 存储技术

### RAM 技术

| | 晶体管数/bit | 访问时间 | 需要刷新 | 需要 EDC | 成本 | 应用 |
|--|-------------|---------|----------|----------|------|------|
| SRAM | 6 or 8 | 1x | No | Maybe | 100x | 缓存 |
| DRAM | 1 | 10x | Yes | Yes | 1x | 内存、帧缓冲区 |

### DRAM 组织

d x w DRAM：总计 d*w 比特，组织为 d 个超级单元（supercells），每个 w 比特。

- **RAS**（Row Access Strobe）：选择行，将整行复制到行缓冲区
- **CAS**（Column Access Strobe）：选择列，将超级单元从缓冲区复制到数据线

### 存储趋势

| 指标 | 1985 | 2015 | 变化 |
|------|------|------|------|
| DRAM $/MB | 880 | 0.02 | 44,000x |
| DRAM 典型大小 (MB) | 0.256 | 16,000 | 62,500x |
| 磁盘 $/GB | 100,000 | 0.03 | 3,333,333x |
| CPU 时钟 (MHz) | 6 | 3,000 | 500x |

### 磁盘

磁盘访问时间主要由寻道时间（seek time）和旋转延迟（rotational latency）决定，其速度比 DRAM 慢几个数量级。

`T_access = T_avg_seek + T_avg_rotation + T_avg_transfer`

示例（7,200 RPM）：
- T_avg_seek = 9 ms
- T_avg_rotation = 4 ms
- T_avg_transfer = 0.02 ms
- T_access = 13.02 ms

> SRAM 访问时间约 4 ns/doubleword，DRAM 约 60 ns。磁盘比 SRAM 慢约 40,000 倍，比 DRAM 慢约 2,500 倍。

### 非易失性存储器（Flash / SSD）

- 数据以**页**为单位进行读取和写入（4 KB 到 512 KB）
- 一个页只有在其所属的**块**被擦除后才能被写入
- 每个块在经历约 10,000 次重复擦写后会磨损失效
- 顺序访问速度远高于随机访问速度
- 通过"磨损均衡"机制延长寿命

### SSD vs 机械硬盘

**SSD 优点**：无机械运动部件，速度更快、功耗更低、更坚固耐用
**SSD 缺点**：存在磨损失效的可能，每字节成本更高

**机械硬盘适用场景**：大容量存储（视频、超大规模数据集、数据库）、低成本存储

## 小结

- CPU、内存与大容量存储之间的速度差距持续扩大
- 精心编写的程序具有局部性特性
- 基于缓存的存储层次结构通过利用局部性来弥合性能鸿沟
- 闪存的发展速度已超过其他所有内存和存储技术（其关键优势在于能够将存储单元在三维空间中堆叠）
