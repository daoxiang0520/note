# 机器级过程调用 (Machine-Level Procedures)

## 调用栈机制 (Call Stack)

### call 指令
- 将**返回地址**（call 下一条指令的地址）压栈
- 跳转到目标函数的起始地址

### ret 指令
- 从栈顶弹出地址
- 跳转到该地址继续执行

> **核心机制**：`call` 压入返回地址，`ret` 弹出返回地址，二者配合实现函数调用与返回。

### 栈帧结构

```
        高地址
        +------------------+
        | 调用者栈帧        |
        +------------------+
        | 返回地址          |  ← call 压入
        +------------------+
        | 被调用者栈帧       |
        |   - 保存的寄存器   |
        |   - 局部变量       |
        |   - 参数构造区     |
        +------------------+  ← %rsp
        低地址
```

## 参数传递 (Argument Passing)

### 前 6 个参数（寄存器传递）

| 参数序号 | 寄存器 | 说明 |
|---------|--------|------|
| 第 1 个 | `%rdi` | 函数第一个参数 |
| 第 2 个 | `%rsi` | 函数第二个参数 |
| 第 3 个 | `%rdx` | 函数第三个参数 |
| 第 4 个 | `%rcx` | 函数第四个参数 |
| 第 5 个 | `%r8` | 函数第五个参数 |
| 第 6 个 | `%r9` | 函数第六个参数 |

### 第 7 个及之后的参数（栈传递）

- 参数 7、8... 按从右到左的顺序压入**调用者的栈帧**
- 在 x86-64 中，前 6 个整数/指针参数始终通过寄存器传递

---

## 寄存器保存约定 (Register Conventions)

| 寄存器 | 类型 | 用途 |
|--------|------|------|
| `%rax` | 调用者保存 (Caller-saved) | 返回值 |
| `%rbx` | 被调用者保存 (Callee-saved) | — |
| `%rcx` | 调用者保存 | 第 4 个参数 |
| `%rdx` | 调用者保存 | 第 3 个参数 |
| `%rbp` | 被调用者保存 | — |
| `%rsp` | 被调用者保存 | 栈指针 |
| `%rsi` | 调用者保存 | 第 2 个参数 |
| `%rdi` | 调用者保存 | 第 1 个参数 |
| `%r8` | 调用者保存 | 第 5 个参数 |
| `%r9` | 调用者保存 | 第 6 个参数 |
| `%r10` | 调用者保存 | — |
| `%r11` | 调用者保存 | — |
| `%r12` | **被调用者保存** | — |
| `%r13` | **被调用者保存** | — |
| `%r14` | **被调用者保存** | — |
| `%r15` | **被调用者保存** | — |

### 调用者保存 (Caller-saved / Call-clobbered)
- 调用者在 `call` 之前必须保存这些寄存器的值（如果需要的话）
- 被调用者可以自由修改

### 被调用者保存 (Callee-saved / Call-preserved)
- 被调用者若使用这些寄存器，必须先保存旧值，返回前恢复
- 调用者可放心使用，值会在 `call` 后保持不变

---

## 局部变量 (Local Variables)

### 未取地址的局部变量
- 通常直接存储在**寄存器**中
- 例如 `int local = -1;` 可能直接存在 `%eax` 中

### 取地址的局部变量
- 必须存储在**栈内存**中，因为寄存器没有内存地址
- 编译器会通过 `sub $N, %rsp` 在栈上分配空间

---

## 尾调用优化 (Tail Call Optimization)

当函数的最后操作是调用另一个函数并直接返回其结果时，编译器可以进行优化：

```c
int returnOneOpt(void) {
    return returnOne();  // 尾调用
}
```

**优化方式**：
- 不创建新的栈帧
- 直接复用当前栈帧
- 使用 `jmp` 代替 `call`（跳转到目标函数，目标函数 `ret` 时直接返回到原调用者）

> 尾调用优化节省了栈空间，在递归函数中尤为重要。

---

## 递归 (Recursion)

递归函数的调用与返回遵循同样的栈帧规则：

```c
int mrec(int n) {
    if (n <= 1)
        return 1;
    return n * mrec(n - 1);
}
```

- 每次递归调用都创建一个新的栈帧
- 每层递归的局部变量和返回地址都保存在栈上
- 栈空间耗尽会导致**栈溢出**（Segmentation Fault）

---

## 补充：字符串在内存中的表示

- GDB 命令 `x/s` 用于显示 C 字符串（以 `\0` 终止的字符序列）
- 字符串本质是字符数组，存储在内存中连续的位置

## 总结

| 概念 | 要点 |
|------|------|
| **call/ret** | call 压入返回地址，ret 弹出并跳转 |
| **参数传递** | 前 6 个用寄存器，7+ 用栈 |
| **寄存器约定** | 调用者保存 vs 被调用者保存 |
| **局部变量** | 未取地址用寄存器，取地址用栈 |
| **尾调用优化** | jmp 代替 call，复用栈帧 |
| **递归** | 每次调用创建新栈帧 |
