# 异常控制流：异常与进程

## 控制流

处理器只做一件事：读取一条指令，执行（解释）该指令，准备下一条指令的地址，重复上述过程。这些指令序列构成了 CPU 的**控制流**。

默认按照顺序执行控制流，可以通过控制指令（jx, call, ret 等）转移控制。

## 改变控制流

到目前为止，有两种机制可以正常地改变控制流：
- 跳转和分支
- 调用和返回

但对于构建实用的系统还不足够，需要**异常控制流**机制来响应系统状态变化：
- 切换到内核代码
- 数据从磁盘或网络适配器到达
- 指令发生除零
- 用户按下 Ctrl-C
- 系统定时器到时

## 异常控制流

发生在计算机系统的各个层次：

| 层次 | 机制 | 实现方式 |
|------|------|----------|
| 低层次 | 异常 | 硬件和操作系统软件结合 |
| 中层次 | 进程上下文切换 | 操作系统软件和硬件定时器 |
| 中层次 | 信号 | 操作系统软件 |
| 高层次 | 非本地跳转 | C 运行时库实现 |

## 异常

**异常**是指在响应某些事件（处理器状态变化）时，控制流被转移到操作系统内核的过程。内核是操作系统中常驻内存的部分。

### 异常表

每种类型的事件都有一个唯一的**异常号** k，作为异常表中的索引（又称中断向量）。每次发生异常 k 时，都会调用对应的处理程序 k。

### 异常的分类

#### 异步异常（中断）—— 线程外部的打断

由处理器外部事件触发：

- **周期性外部事件**：定时器中断（进程调度与时间片管理、系统时钟维护）
- **非周期性外部事件**：外部设备的 I/O 中断（用户按下 Ctrl-C、网络数据包到达、磁盘数据到达）

处理程序返回到"下一条"指令。

#### 同步异常 —— 线程内部的障碍或陷阱

由执行指令时发生的事件引发：

| 类型 | 特征 | 示例 | 返回行为 |
|------|------|------|----------|
| **陷阱 (trap)** | 有意的 | 系统调用、断点陷阱 | 返回到下一条指令 |
| **故障 (fault)** | 无意的，但可能是可恢复的 | 缺页异常（可恢复）、保护错误（不可恢复） | 重新执行当前指令或终止 |
| **终止 (abort)** | 无意的，且不可恢复 | 非法内存地址、非法指令、奇偶校验错 | 终止当前程序 |

## 系统调用

系统调用是操作系统内核提供给应用程序的核心接口，完成了从用户态到内核态的切换。

### x86-64 系统调用

每个系统调用都有一个唯一的 ID 号：

| 编号 | 名称 | 描述 |
|------|------|------|
| 0 | read | 读文件 |
| 1 | write | 写文件 |
| 2 | open | 打开文件 |
| 3 | close | 关闭文件 |
| 4 | stat | 获得文件信息 |
| 57 | fork | 创建进程 |
| 59 | execve | 执行一个程序 |
| 60 | _exit | 终止进程 |
| 62 | kill | 发送信号 |

**调用约定**：
- `%rax` 包含系统调用号
- `%rdi`, `%rsi`, `%rdx`, `%r10`, `%r8`, `%r9` 包含最多 6 个参数
- 返回值在 `%rax` 中，负值表示错误

```asm
mov $0x2,%eax    # open is syscall #2
syscall          # return value in %rax
cmp $0xfffffffffffff001,%rax  # check for error
```

### 系统调用 vs 库函数

| | 系统调用 | 库函数 |
|--|----------|--------|
| 定义 | OS 内核提供的接口 | 编程语言提供的库函数 |
| 示例 | `open`, `read` | `fopen`, `printf` |
| man 手册 | man 2 | man 3 |

### 故障示例：缺页异常

```c
int a[1000] = {...};
main() {
    a[500] = 13;  // 触发缺页异常，内核将页从磁盘复制到内存，然后重新执行
}
```

### 故障示例：非法的内存引用

```c
int a[1000];
main() {
    a[5000] = 13;  // 检测到无效地址，向进程发送 SIGSEGV 信号
}
```

## 进程

**进程**是一个正在运行的程序的实例，为每个程序提供两个关键抽象：

1. **逻辑控制流**：看起来像程序独占地使用 CPU（由上下文切换提供）
2. **私有地址空间**：看起来像程序独占地使用主存（由虚拟内存提供）

### 多进程

- 单个处理器同时执行多个进程，进程交错执行（多任务处理）
- 地址空间由虚拟内存系统管理
- 非执行进程的寄存器值保存在内存中

### 并发 vs 并行

- **并发**：两个进程的执行在时间上有重叠
- **并行**：物理上的同时发生

### 上下文切换

进程由常驻内存的、共享的操作系统代码块（称为**内核**）管理。控制流通过**上下文切换**从一个进程转移到另一个进程。内核不是一个独立的进程，而是作为某个现有进程的一部分运行。

## 总结

- **异常**：需要非标准控制流的事件，外部生成（中断）或内部生成（陷阱和故障）
- **进程**：系统在任意给定时间有多个活动进程，但同一时间在单个核心上只能有一个进程执行。每个进程看起来都可以完全控制处理器和私有内存空间。
