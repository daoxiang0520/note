# 虚拟存储器的概念

## 为什么使用虚拟存储

- **高效使用主存**：把 DRAM 用作部分虚拟地址空间的缓存
- **简化内存管理**：每个进程都获得相同的统一线性地址空间
- **隔离地址空间（保护）**：一个进程不能干扰另一个进程的内存，用户程序不能访问特权内核信息和代码

## 虚拟寻址系统

所有现代服务器、笔记本电脑和智能手机都使用虚拟寻址系统。MMU（Memory Management Unit）负责将虚拟地址转换为物理地址。

```
CPU -> VA -> MMU -> PA -> 主存
```

## 地址空间

- **虚拟地址空间**：N = 2^n 个虚拟地址的集合 {0, 1, 2, 3, ..., N-1}
- **物理地址空间**：M = 2^m 个物理地址的集合 {0, 1, 2, 3, ..., M-1}

## 地址转换

MAP: V -> P U {∅}

- 如果数据在物理内存中：MAP(a) = a'（物理地址）
- 如果数据不在物理内存中：MAP(a) = ∅（缺页故障）

## 页表

页表是页表项（PTE）的数组，用虚拟页号索引，将虚拟页映射到物理页。

- 页表是每个进程驻留在内存中的内核数据结构
- 如果页在内存中：PTE 存储对应的物理页号和状态信息
- 如果页不在内存中：PTE 存储磁盘上的位置

```
虚拟地址: [VPN][VPO]
物理地址: [PPN][PPO]
```

### 页命中

访问的虚存字在物理内存中（DRAM 缓存命中）。

### 缺页故障

访问的虚存字不在物理内存中（DRAM 缓存缺失）。

**缺页处理流程**：
1. MMU 访问页表，发现有效位为 0
2. MMU 触发缺页故障异常
3. 处理程序确认驱逐页（如果 dirty 位为 1，复制回磁盘）
4. 处理程序从磁盘读取新页，更新页表
5. 处理程序返回，重启引发故障的指令

> 只有在缺失时才将页复制到 DRAM 的方式称为**按需分页**。

### 分配页

分配虚存的一个新页只是建立映射，只有第一次访问才会真正按需加载到内存。

## 多级页表

### 线性页表的问题

32 位虚拟地址，4KB 页表，每个进程需要 2^20 个虚拟页，PTE 4 字节 => 4MB 页表。64 位地址空间则完全不可行。

### 多级页表解决方案

使用多级页表，类似稀疏矩阵的表示。一级表常驻内存，二级表按需分配。

```
页表基址寄存器 (PTBR) -> 一级页表 -> 二级页表 -> ... -> k 级页表 -> PPN
```

RISC-V 使用四级页表：48 位虚拟地址，映射到 40 位物理地址。

## TLB（Translation Look-aside Buffer）

TLB 是 MMU 中的一个小型组相联硬件缓存，专门存储少量页的完整页表项，将 VPN 映射到 PPN。

- 典型大小：16-512 页表项
- 命中延迟：0.5-1 个时钟周期
- 失效惩罚：10-100 个时钟周期
- 失效率：0.01%-1%

### TLB 命中

消除一次内存访问（多级页表则消除多次）。

### TLB 缺失

导致额外的内存访问（PTE）。TLB 缺失很少发生，因为页表也有很好的局部性。

## 虚拟内存用于内存保护

给页表项增加权限位（READ, WRITE, EXEC），每次访问时 MMU 检查这些位。

- 只有 OS 可以修改页表
- 特权模式（内核模式）才能访问页表和相关状态

## DRAM 缓存结构

- DRAM 大约比 SRAM 慢 10 倍，磁盘比 DRAM 慢 10,000 倍
- 大的页容量：一般为 4KB，有时是 4MB
- 全相联：每个虚拟页都可以放到任意物理页中
- 回写而非通写
- 偏好 LRU 替换算法

## 局部性

虚拟存储器看起来效率极低，但由于局部性它是有效的。任何时间点，程序通常会访问一组活跃的虚拟页，称之为**工作集**。

- 如果工作集大小 < 主存容量：强制缺失后表现良好
- 如果 SUM(工作集大小) > 主存容量：**抖动**，不断交换页导致性能崩溃

## 总结

- **程序员视角**：每个进程都有自己的私有线性地址空间，不会被其他进程破坏
- **系统视角**：通过缓存虚存页来高效使用内存，高效仅仅是因为局部性
