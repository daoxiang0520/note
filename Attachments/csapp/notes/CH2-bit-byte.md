# Bits, Bytes, and Integers

> CMU 15-213: Introduction to Computer Systems, 2nd and 3rd Lectures, Sep. 3 and Sep. 8, 2015

## 信息以位表示

### Everything is Bits

- 每个 bit 是 0 或 1，可以用各种方式来编码和解释 bit 序列
- 计算机确定要做什么（指令），可以表示数字、集合、字符串等
- **为什么用 bit？** 基于电子电路的实现，二值稳态更容易保持（或存储），可以更可靠地在有噪声的线路上传送

### 数字的二进制表示

- 十进制整数转二进制："除2取余，逆序排列"
- 十进制小数转二进制："乘2取整，顺序排列"
- 有限十进制小数在二进制中可能是无限小数（如 0.1）

### 位与字节（Bit and Byte）

- 1 字节 = 8 bits
- 二进制：`00000000_2` to `11111111_2`
- 十进制：`0_10` to `255_10`
- 十六进制：`00_16` to `FF_16`
- C/C++ 字面量：`0b10`（二进制）、`10`（十进制）、`0x10`（十六进制）

| Hex | Decimal | Binary |
|------|---------|--------|
| 0 | 0 | 0000 |
| 1 | 1 | 0001 |
| 2 | 2 | 0010 |
| ... | ... | ... |
| F | 15 | 1111 |

### C 数据类型大小

## 位级别的操作

### 布尔代数

由 **George Boole** 在 19 世纪创立，将逻辑代数化：

- **And** `A & B` — 当 A=1 且 B=1 时为 1
- **Or** `A | B` — 当 A=1 或 B=1 时为 1
- **Not** `~A` — 当 A=0 时为 1
- **Exclusive-Or (Xor)** `A ^ B` — 当 A=1 或 B=1（但不同时为 1）时为 1

> 1930 年代，Claude Shannon 在其硕士论文中证明了布尔代数可以用于分析并设计开关电路。

### 位向量及运算

```
  01101001
& 01010101
----------
  01000001

  01101001
| 01010101
----------
  01111101

  01101001
^ 01010101
----------
  00111100

~ 01010101 = 10101010
```

### 用位向量表示集合

宽度为 w 的位向量可以用来表示集合 `{0, ..., w-1}` 的子集：如果 `j ∈ A`，则 `a_j = 1`

- `A: 01101001` => `{0, 3, 5, 6}`
- `B: 01010101` => `{0, 2, 4, 6}`
- `A & B` 交集 => `{0, 6}`
- `A | B` 并集 => `{0, 2, 3, 4, 5, 6}`
- `A ^ B` 对称差 => `{2, 3, 4, 5}`
- `~B` 补集 => `{1, 3, 5, 7}`

### C 语言中的位运算

| 运算符 | 含义 |
|--------|------|
| `&` | 按位与 |
| `|` | 按位或 |
| `~` | 按位取反 |
| `^` | 按位异或 |

适用于所有整数类型：`long`、`int`、`short`、`char`、`unsigned`

**例子（char 类型）：**

```c
~0x41 -> 0xBE
~0x00 -> 0xFF
0x69 & 0x55 -> 0x41
0x69 | 0x55 -> 0x7D
```

### C 语言中的逻辑运算

| 运算符 | 含义 |
|--------|------|
| `&&` | 逻辑与 |
| `||` | 逻辑或 |
| `!` | 逻辑非 |

- 将 0 值操作数视为 "False"，所有非 0 值操作数视为 "True"
- 运算结果只能是 0（False）或 1（True）
- 支持**提前结束运算**（短路求值）

例子：

```c
!0x41 -> 0x00
!0x00 -> 0x01
!!0x41 -> 0x01
0x69 && 0x55 -> 0x01
0x69 || 0x55 -> 0x01
if (p && *p)  // 可以避免空指针访问
```

### 位运算 vs 逻辑运算

| 位运算 | 逻辑运算 |
|--------|----------|
| `&`, `|`, `~`, `^` | `&&`, `||`, `!` |
| 逐位操作 | 整体判断 0/非0 |
| 输出与操作数同宽 | 输出为 0 或 1 |
| 不短路 | 短路求值 |

## 整数表示

### 无符号数与有符号数

- **无符号数**：使用标准二进制编码，范围 `[0, 2^w - 1]`
- **有符号数**：使用补码编码，范围 `[-2^{w-1}, 2^{w-1} - 1]`

### 类型转换

- 无符号和有符号之间的转换保持位模式不变，只改变解释方式
- 扩展：零扩展（无符号） vs 符号扩展（有符号）
- 截断：取低 k 位

### 整数运算

- 加法、减法、乘法、移位
- 溢出检测和处理
- 补码与非补码

## 内存中的表示

### 字节序

- **小端**（Little Endian）：最低有效字节在最低地址（如 x86-64）
- **大端**（Big Endian）：最高有效字节在最低地址

### 指针和字符串

- 指针存储内存地址
- 字符串以 null 结尾的字符数组形式存储

---

## 总结

- 所有信息都以 bit 表示
- 位运算提供低层次的数据操作能力
- 整数有多种表示方式，需要理解转换和运算规则
- 内存中的表示方式影响数据解释
