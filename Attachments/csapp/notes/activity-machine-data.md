# 机器级数据布局 (Data in Memory)

## 局部变量的存储

### 未取地址的局部变量
- 直接存放在**寄存器**中
- 例如 32 位 `int local` 可能存放在 `%eax`（`%rax` 的低 32 位）

### 取地址的局部变量
- 必须分配在**栈内存**中（因为寄存器没有内存地址）
- 编译器在栈上分配空间：`sub $N, %rsp`

```c
int returnOne(void) {
    int local = -1;         // 可能直接存在寄存器中
    return abs(local);
}

int returnOneTwo(void) {
    int local = -1;
    return absp(&local);    // 取地址 -> 必须存栈上
}
```

> 如果函数需要用指针访问局部变量（通过 `&` 运算符），编译器将变量从寄存器"溢出"（spill）到栈上。

## 数组 (Arrays)

### 数组的步长 (Stride)

```c
int courses[4] = {0x15213, 0x15513, 0x18213, 0x18600};
int getNth(int *arr, size_t index) {
    return arr[index];
}
```

| 数组类型 | 元素大小 | 步长 |
|----------|---------|------|
| `int[]` | 4 字节 | 4 |
| `char[]` | 1 字节 | 1 |
| `long[]` | 8 字节 | 8 |

在汇编层，数组访问通过**基址 + 偏移量**实现：
```asm
# arr 在 %rdi, index 在 %rsi
movl (%rdi, %rsi, 4), %eax   # 取 arr[index]，步长 4 编码在指令中
```

### 字符串与 x/s

字符串是字符数组 + 空字符终止符（`\0`）。

- GDB 中 `x/s <addr>` 从该地址开始逐字节打印，直到遇到 `\0`
- `x/12bx <addr>` 以字节数组形式显示前 12 个字节

## 结构体 (Structs)

### 简单结构体

```c
struct course {
    int cs_ugrad;
    int cs_grad;
    int ece_ugrad;
    int ece_grad;
};
```

所有成员大小相同，布局完全按声明顺序连续排列。

### 混合类型结构体与对齐 (Alignment)

编译器会在成员之间和结构体末尾插入**填充字节 (padding)** 以满足对齐要求：

```c
struct increasing {
    char a;      // 1 字节，偏移 0
    short b;     // 2 字节，偏移 2（填充 1 字节）
    int c;       // 4 字节，偏移 4
    long d;      // 8 字节，偏移 8
};
```

#### 对齐规则

| 类型 | 对齐要求 | 大小 |
|------|---------|------|
| `char` | 1 字节 | 1 |
| `short` | 2 字节 | 2 |
| `int` | 4 字节 | 4 |
| `long` / 指针 | 8 字节 | 8 |

> **基本原则**：类型的地址必须是其对齐大小的倍数。结构体的总大小必须是其最宽成员对齐大小的倍数。

### 字段重排的影响

```c
// 更好的布局：按对齐要求从大到小排列
struct increasing {
    long d;      // 8 字节
    int c;       // 4 字节
    short b;     // 2 字节
    char a;      // 1 字节
    // 总共 15 字节，对齐到 16
};

// 较差的布局：按类型从小到大排列（产生更多填充）
struct rearranged {
    char a;      // 1 字节
    // 填充 7 字节
    long b;      // 8 字节，偏移 8
    short c;     // 2 字节，偏移 16
    // 填充 4 字节
    int d;       // 4 字节，偏移 20
    // 总共 24 字节
};
```

> **经验法则**：将结构体成员按**对齐要求从大到小**排列，可以最小化填充，减少内存浪费。

## 结构体数组 (Arrays of Structs)

```c
struct pair {
    int large;    // 4 字节
    char small;   // 1 字节
    // 填充 3 字节
};
// sizeof(struct pair) = 8

struct pair pairs[2] = {
    {0xabababab, 0x1},
    {0xcdcdcdcd, 0x2}
};
```

- 每个元素的大小 = 结构体的大小（含填充）
- 数组步长 = `sizeof(struct pair)`
- 填充确保数组元素的正确对齐

### 结构体内含数组

```c
struct triple {
    short large[2];  // 4 字节
    char small;      // 1 字节
    // 填充 3 字节
};
```

- 对齐规则仍然是按最大成员的对齐要求
- 结构体总大小为对齐的倍数

## 二维数组 (2D Arrays)

C 中有两种实现二维数组的方式：

### 方式 1：嵌套数组 (Nested Arrays)

```c
int8_t nested[2][3] = {
    {0x00, 0x01, 0x02},
    {0x10, 0x11, 0x12}
};
```

- 以**行优先**顺序连续存储在内存中
- 内层步长 = 3 字节（每行包含 3 个元素）
- 外层步长 = 6 字节（整个数组）
- 访问函数：`int8_t access(int8_t (*arr)[3], size_t row, size_t column)`
- 内层步长**在编译时固定**（编码在指令中）

### 方式 2：多级数组 (Multilevel Arrays)

```c
int8_t first[3]  = {0x00, 0x01, 0x02};
int8_t second[3] = {0x10, 0x11, 0x12};
int8_t *multilevel[2] = {first, second};
```

- 外层数组存储的是**指针**（每个 8 字节）
- 内层数组可以独立分配在内存的不同位置
- 访问函数：`int8_t accessMultilevel(int8_t **arr, size_t row, size_t column)`
- 需要两次内存访问（一次取指针，一次取数据）
- **灵活**：内层数组可以有不同的长度

### 两种方式的对比

| 特性 | 嵌套数组 | 多级数组 |
|------|---------|---------|
| 内存布局 | 连续的行优先 | 指针数组 + 独立行 |
| 步长 | 编译时固定 | 运行时确定 |
| 访问次数 | 1 次（基址 + 偏移） | 2 次（解引用指针再访问） |
| 灵活性 | 每行长度必须相同 | 每行长度可以不同 |
| 使用场景 | 固定大小矩阵 | 不规则结构（如字符串数组） |

## 字节序 (Endianness)

字节序定义了多字节整数在内存中的字节排列顺序。

### Little-Endian (x86-64 采用)

低地址存放**最低有效字节**（LSB first）。

```c
int courses[4] = {0x15213, ...};
// 大端视角：0x00015213
// 实际存储（小端）：
// 地址递增 → 13 52 01 00
```

### Big-Endian

低地址存放**最高有效字节**（MSB first）。

### 小端序的特点

| 方面 | 说明 |
|------|------|
| **直观性** | 人类直接阅读不便（字节顺序"颠倒"） |
| **类型转换** | 窄化转换（如 `long*` → `int*`）不需要移动指针 |
| **地址运算** | `(int)p` 和 `p` 指向同一地址，低字节自然对齐 |

> x86-64 是小端架构。网络协议通常使用大端（网络字节序），因此需要 `htonl`/`ntohl` 等函数转换。

## 总结

| 主题 | 关键点 |
|------|--------|
| **局部变量** | 未取地址存寄存器，取地址存栈 |
| **数组步长** | 等于元素大小，编码在汇编指令中 |
| **结构体对齐** | 自然对齐 + 填充，从大到小排列字段可减小体积 |
| **二维数组** | 嵌套数组连续存储，多级数组通过指针间接访问 |
| **字节序** | x86-64 小端，低地址存低位字节 |
